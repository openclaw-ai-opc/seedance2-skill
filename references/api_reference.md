# Seedance 2.0 API 参考文档

## 基础信息

- **API 基础URL**: `https://dzwlai.com/apiuser/_open/ai/task/api`
- **认证方式**: Header 中的 `x-token` 字段
- **数据格式**: JSON

## 接口列表

### 1. 提交视频生成任务

**接口地址**: `POST /seedance/gene2video`

**请求头**:
```
Content-Type: application/json
x-token: <YOUR_TOKEN>
```

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| modelName | string | 是 | 模型名称，当前支持: `seedance-2-0-fast` |
| prompt | string | 是 | 视频描述提示词，支持中英文，建议50-200字 |
| duration | number | 否 | 视频时长，单位秒，默认5，范围1-10 |
| resolution | string | 否 | 分辨率，默认"480p"，可选: "480p", "720p" |
| aspect_ratio | string | 否 | 宽高比，默认"16:9"，可选: "16:9", "9:16", "1:1" |
| generate_audio | boolean | 否 | 是否生成音频，默认false |
| watermark | boolean | 否 | 是否添加水印，默认true |
| web_search | boolean | 否 | 是否启用网络搜索增强，默认false |
| return_last_frame | boolean | 否 | 是否返回最后一帧图片，默认false |
| callbackUrl | string | 否 | 任务完成回调URL，可选 |

**响应示例**:

成功:
```json
{
  "code": 200,
  "msg": "您已成功提交创作任务，目前剩余并发任务数为3",
  "data": {
    "id": "1909553892857782274",
    "status": "create"
  }
}
```

失败:
```json
{
  "code": 400,
  "msg": "参数错误：prompt不能为空"
}
```

### 2. 查询任务状态

**接口地址**: `GET /getState`

**请求头**:
```
x-token: <YOUR_TOKEN>
```

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| ids | string | 是 | 任务ID，多个用逗号分隔 |

**响应示例**:

处理中:
```json
{
  "code": 200,
  "msg": "success",
  "data": [
    {
      "id": "1909553892857782274",
      "status": "processing",
      "progress": 45
    }
  ]
}
```

成功:
```json
{
  "code": 200,
  "msg": "success",
  "data": [
    {
      "id": "1909553892857782274",
      "status": "success",
      "videoUrl": "https://cdn.example.com/video/xxx.mp4",
      "duration": 5,
      "resolution": "480p",
      "createdAt": "2025-01-15T10:30:00Z",
      "completedAt": "2025-01-15T10:35:00Z"
    }
  ]
}
```

失败:
```json
{
  "code": 200,
  "msg": "success",
  "data": [
    {
      "id": "1909553892857782274",
      "status": "failed",
      "error": "内容审核未通过"
    }
  ]
}
```

## 任务状态说明

| 状态 | 说明 |
|------|------|
| create | 任务已创建，等待处理 |
| pending | 任务排队中 |
| processing | 正在生成视频 |
| success | 生成成功，可获取视频URL |
| failed | 生成失败，查看error字段 |

---

## 错误码说明

| 错误码 | 说明 |
|--------|------|
| 200 | 成功 |
| 400 | 参数错误 |
| 401 | 认证失败，token无效或过期 |
| 403 | 权限不足 |
| 429 | 请求过于频繁或并发任务已满 |
| 500 | 服务器内部错误 |

## 提示词编写指南

### 结构化提示词

推荐的提示词结构：

```
[风格] + [场景描述] + [主体动作] + [镜头运动] + [细节补充]
```

### 示例

**写实风格**:
```
写实风格，晴朗的蓝天之下，一大片白色的雏菊花田，
镜头逐渐拉近，最终定格在一朵雏菊花的特写上，
花瓣上有几颗晶莹的露珠，阳光透过花瓣
```

**动漫风格**:
```
日系动漫风格，春天的樱花树下，
穿着校服的少女轻轻接住飘落的花瓣，
镜头从侧面缓缓推进，背景是蓝天和飘舞的樱花
```

**电影感**:
```
电影质感，黄昏时分的城市天际线，
镜头从高空俯瞰整个城市，缓缓向前推进，
夕阳将天空染成橙红色，建筑物的灯光逐渐亮起
```

### 关键词建议

**风格关键词**:
- 写实风格、动漫风格、电影质感、油画风格、水彩风格

**镜头运动**:
- 镜头拉近、镜头推远、镜头平移、俯拍、仰拍、跟拍

**氛围描述**:
- 晴朗、阴天、黄昏、夜晚、雾气、阳光明媚

## 最佳实践

### 1. 并发控制

- 注意剩余并发数（响应msg中会提示）
- 避免同时提交大量任务
- 建议串行或小批量提交

### 2. 轮询策略

- 推荐轮询间隔: 5-10秒
- 最大等待时间: 10分钟（复杂场景可能更长）
- 使用指数退避处理网络错误

### 3. 错误处理

- 429错误: 等待后重试
- 401错误: 检查token有效性
- 任务失败: 检查提示词是否合规

### 4. 资源管理

- 及时下载生成的视频
- 视频URL可能有时效性
- 建议本地保存后删除远程任务

---

## 限制说明

- 单次生成时长: 1-10秒
- 并发任务数: 根据套餐限制
- 提示词长度: 建议50-200字
- 支持语言: 中文、英文
